# Image Guidelines

## Purpose

This document explains how website-ready images should be organized and named for Suraiya Anvery's portfolio website.

## Master Images vs Website Images

Keep original full-quality photos outside the website repository in a separate master source image folder on the computer.

Do not dump raw Google Drive folders directly into the repo.

The repo should only contain selected images intended for the website.

## Website Image Location

Selected website-ready images should live in:

assets/images/work/

The current category folders are:

- assets/images/work/quilts/
- assets/images/work/bags/
- assets/images/work/pouches/
- assets/images/work/table-runners/
- assets/images/work/everyday-essentials/
- assets/images/work/exhibitions/

## Work Folder Naming

Each work/item should have its own lowercase kebab-case folder.

Example:

assets/images/work/quilts/2880-squares/

## Image Naming

Each work/item folder should eventually use:

- main.jpg
- detail-1.jpg
- detail-2.jpg
- detail-3.jpg
- detail-4.jpg

Use more detail images only when needed.

## Image Roles

main.jpg:
- Used for category grids
- Used for homepage/overview cards
- Used for featured previews

detail images:
- Used on individual work/item pages
- Used for close-ups, alternate angles, process details, or contextual photos

## Naming Rules

- Use lowercase filenames.
- Avoid spaces.
- Use hyphens if descriptive filenames are needed.
- Prefer jpg for photos unless transparency is needed.
- Avoid extremely large unoptimized image files in the repo when possible.

## Future Optimization

Before deployment, selected images may be resized and compressed for performance.

Potential future sizes:
- Main/card images: around 1600px wide maximum
- Detail images: around 2000px wide maximum if needed
- Thumbnails: smaller versions may be created later if performance requires it
